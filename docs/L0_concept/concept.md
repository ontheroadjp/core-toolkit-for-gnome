# コンセプト

## 目的

このリポジトリは、特定の1台のマシン（ホスト名 `t480s`、Lenovo ThinkPad T480s 上の
GNOME + Ubuntu 24.04 環境。根拠: `uname -a` のホスト名 `t480s` と
`6.17.0-35-generic ... 24.04.1-Ubuntu`、リポジトリ内のスクリプト名
`t480s.sh` / `t480s_apps.sh`）を、再現可能な手順でセットアップ・運用するための
個人用ツールキットである。

汎用フレームワークやアプリケーションではなく、以下3種類のファイル群を
バージョン管理することが目的:

1. GNOME デスクトップの挙動を `gsettings` で設定するスクリプト
   （`t480s.sh`）
2. マシン初期セットアップ時に必要なパッケージ・CLI ツールを
   `apt` / `curl` / `wget` / `npm` 経由で導入するスクリプト
   （`t480s_apps.sh`）
3. ホームディレクトリにシンボリックリンクとして配置される設定ファイル・
   ユーティリティスクリプト（`.config/alacritty/**`, `.local/bin/gnome-overview-toggle`）

## 解決する問題

OS再インストールやマシン更新時に、ターミナルの見た目・キーバインド・
GNOME の挙動・必要なCLIツール群を手作業で再構築する手間を、
スクリプト実行とシンボリックリンクの再作成だけで済ませられるようにする。

根拠: `~/.config/alacritty` および
`~/.local/bin/gnome-overview-toggle` が、実際にこのリポジトリ内の
パスへのシンボリックリンクとして存在することを `readlink -f` で確認した
（システム状態の直接観測。リポジトリ内にこれを自動化するインストーラ
スクリプトは存在しない — 未確認事項として後述）。

## 対象ユーザー

リポジトリ所有者本人（シングルユーザー、シングルマシン想定）。
`t480s_apps.sh` 内のコメントに `MBP15` 向けの代替フォント設定が
`.config/alacritty/alacritty.toml:38-43` にコメントアウトされた状態で残っており、
過去に複数マシン運用を想定していた痕跡があるが、現在アクティブな設定・
スクリプトは T480s 向けのみである（未確認: MBP15 用スクリプトが
別途存在したかどうかは git 履歴からは追えない。本リポジトリは
本コマンド実行時点でコミット履歴が0件のため、過去の状態は確認不能）。

## 設計上の制約

- 単一マシン・単一ユーザーの個人ツールという前提で、抽象化や設定の
  汎用化は行われていない（例: `t480s.sh` のバッテリー充電閾値
  `t480s.sh:53-54` は T480s 固有のしきい値をハードコードしている）。
- `sudo` を要するコマンドが複数のスクリプトに直接記述されており
  （`t480s.sh:53-54`、`t480s_apps.sh:9,29,39,68-73` 等）、
  対話的に人間が実行する前提でエラーハンドリングは行われていない。
- CI・自動テストは存在しない（`.github/` 不在、テストファイル不在を確認済み）。
  個人用のシェルスクリプト集であり、実行結果はマシン上で目視確認する運用。
